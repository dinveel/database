UNTIED TRANSPORT COMPANY
База «UTC» представляет собой базу данных для диспетчера транспортной компании, предоставляющей заказчику спецтехнику для проведения различных видов работ на основе субподряда. База содержит списки организаций, сотрудничающих между собой в составе объединенной транспортной компании с перечнем имеющейся в наличии строительной техники.
Диспетчер при поступлении заявки на проведение работ должен определить, кому из подрядчиков можно передать заявку на исполнение. Заявка может быть оформлена в трёх основных видах:
1. Указан вид работ, которые необходимо выполнить. На основании информации из базы данных диспетчер может определить, кто из партнёров обладает необходимым набором техники и передать заявку ближайшему к месту проведения работ подходящему партнёру.
2. Указана конкретная техника, которую необходимо предоставить. На основании информации из базы данных диспетчер может определить, у кого из партнёров свободна необходимая техника и передать ему заявку.
3. Указан вид работ плюс дополнительная техника. На основании информации из базы данных диспетчер подбирает партнёра с необходимым набором техники (набор плюс доп. единицы) и передаёт ему заявку.
База помогает оператору не только получить информацию о наличии свободной техники у субподрядчика, но и сформировать запрос о наличии свободной техники у партнеров субподрядчика для её оперативной переброски на объект, где она понадобилась или кратно увеличить количество техники на объекте для сокращения сроков исполнения работ и минимизации простоя техники. Также в базе имеется информация о среднем расходе топлива для различных видов техники, что позволяет диспетчеру своевременно направить заправщик на объекты.


(1 таблица - subcontractors(субподрядчики))
В этой таблице представлен перечень партнерский организаций, поставляющих строительную технику для проведения работ.
subcontractor - список партнеров, работающих в составе компании.
area_id_len_obl - данный столбец содержит коды районов осуществления деятельности. Каждому партнеру сопоставлен код района, в котором он осуществляет свою деятельность. Для упрощения работы диспетчера, код района сопоставлен коду налоговой инспекции, в которой зарегистрирован партнер. Коды налоговых инспекций уникальны по всей территории РФ и составлены по определённым правилам, что помогает подобрать для совместного выполнения работ партнёров из одного района (все четыре цифры кода) или одного региона (первые две цифры кода), что важно для минимизации расходов на доставку техники.
dispatcher_telephone - поскольку в компаниях-партнёрах количество техники может меняться (ремонт, тех. Обслуживание, расширение парка и т.д.), в БД введен телефон партнеров, по которому можно уточнить информацию и скорректировать базу данных.
id_subcontractor - содержит уникальный код партнера для идентификации в базе данных, поскольку в разных районах организации могут иметь одинаковое название.


(2 таблица - len_obl(районы))
Таблица содержит список районов осуществления деятельности партнеров и соответствующие им коды. Сопоставляет коду района его название, где осуществляется деятельность.
area_id_len_obl -код района, где осуществляется деятельность партнера.
area - название района, где партнер осуществляет свою деятельность.


(3 таблица - subcontractors_technique_amount)
Таблица содержит количество различных видов техники, имеющейся в распоряжении субподрядчика.
id_subcontractor - индивидуальный идентификатор, определяющий субподрядчика.
id_technique - индивидуальный идентификатор вида техники.
amount_of_tech - исходное количество техники (максимальное), которой обладает субподрядчик.

(4 таблица - technique)
Сопоставляет идентификатору вида техники его название и количество и усредненный расход топлива этой техникой.
id_technique - индивидуальный идентификатор вида техники.
techniuqe - вид техники.
Fuel_per_hour - среднее количество топлива, затрачиваемое конкретной техникой за час.

(5 таблица - Contract_subcontractor)
Содержит технический номер контракта и индивидуальный номер партнера, которому был выдан данный контракт.
id_contract - технический номер контракта. В данной таблице используется, для создания связи субподрядчик-контракт.
Id_subcontractor - индивидуальный идентификатор, определяющий субподрядчика, которому выдан для исполнения данный контракт.

(6 таблица - contracts)
Сводная таблица информации о контракте (идентификатор, дата начала и окончания контракта, район работы, стадия работы, идентификатор типа работы, коэффициент контракта)
id_contract - индивидуальный идентификатор, определяющий контракт.
Date_of_begin - дата начала действия контракта.
Date_of_end - дата окончания действия контракта.
Work_area - район, в котором будет выполняться работа.
Stage_of_work - стадия, в которой находится контракт. Контракт может быть в 3х состояниях:
1.’not started’ – контракт был заключен, но дата его исполнения еще не наступила
2.’in progress’ – контракт находится в стадии исполнения
3.’ended’ – контракт был исполнен
Id_type_of_work - индивидуальный идентификатор, определяющий вид работы.
Koef_of_contract - коэффициент контракта. Дает информацию о том, что набор техники должен быть кратно увеличен для сокращения сроков исполнения контракта.

(7 таблица - Type_of_work)
Содержит основные типы работ, выполняемых на строительных объектах.
id_type_of_work - индивидуальный идентификатор, определяющий вид работы.
Type_of_work - вид работы.

(8 таблица - Type_of_work_tech)
Содержит количество техники, которое предположительно потребуется для проведения данного вида работ.
id_type_of_work - индивидуальный идентификатор, определяющий вид работы.
Id_technique - индивидуальный идентификатор вида техники.
Amount_of_tech_is_needed - минимальное количество техники, требуемое для выполнения данного вида работы.
